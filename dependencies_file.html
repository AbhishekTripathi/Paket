The paket.dependencies file

The paket.dependencies file is used to specify rules regarding your application's dependencies.

github forki/FsUnit FsUnit.fs

The syntax looks familiar to users of Ruby's bundler Gemfile. This is intended because it proved to work well for the authors of Paket.

The file specifies that Paket's NuGet dependencies should be downloaded from nuget.org and that we need e.g.:

  • NUnit in version 2.6.3 <= x < 2.7
  • FAKE in version 3.4 <= x < 3.5 as a build tool

Only direct dependencies should be listed. Paket uses this definition to compute a concrete dependency resolution, which also includes indirect dependencies. The resulting dependency graph is then persisted to the paket.lock file.
